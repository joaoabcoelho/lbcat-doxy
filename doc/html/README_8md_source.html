13 Installation:
14 -------------
15 
16 You need to have cmake installed on your machine (this is on the
17 standard repositories and probably already installed on your machine).
18 CMT is not used anymore.
19 
20 Just checkout a new working copy of the software, but requiring a
21 specific branch which is called cmake.
22 
23 Go to the main directory and adapt the setup.sh file. You
24 should have only to modify the CATPATH variable, so that it points to
25 the directory where the software is installed.
26 
27 Type
28 
29 > source setup.sh
30 
31 > cmake .
32 
33 > make all
34 
35 > make install
36 The doxygen documentation can be built with the command
37 
38 > make doc
39 
40 Starting Cat:
41 -------------
42 
43 To start cat you can run either in any place (alias defined in the
44 setup.sh file):
45 
46 > catcmd
47 for the command version
48 
49 > catgui
50 for the gui version.
51 It is advised to create another directory where you start the software,
52 so that all the user files/directory will stand in another place than
53 the source and compilation files.
54 An alias setupCat is useful to launch the command
55 
56 > source setup.sh
57 with the proper path to the source.sh file.
